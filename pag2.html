HTML formatting

Formatação HTML se refere ao uso de elementos e atributos HTML para definir a apresentação visual e a estrutura do conteúdo em uma página da web. Isso inclui a definição de estilos de texto, alinhamento, listas, links, imagens e outros elementos para melhorar a legibilidade e o apelo visual do conteúdo. Aqui estão alguns dos principais elementos e atributos HTML usados para formatação:

  1. Textos

    2. HTML fornece uma variedade de elementos para formatar texto, como h1 a h6 para títulos de diferentes níveis de importância, p para parágrafos, strong para texto em negrito, em para texto em itálico, u para sublinhar texto, s para texto tachado, sup para texto sobrescrito e sub para texto subscrito.

  2. Listas

    3. HTML permite criar listas ordenadas ol e listas não ordenadas ul, bem como listas de definição dl. Dentro desses elementos, você pode usar li para definir itens de lista.

  3. Links

    4. O elemento a é usado para criar links em HTML. Você pode definir o destino do link usando o atributo href e opcionalmente adicionar um texto âncora entre as tags de abertura e fechamento do a.

  4. Imagens

    5. O elemento img é usado para incluir imagens em uma página HTML. Você precisa fornecer o atributo src com o caminho da imagem e opcionalmente os atributos alt para texto alternativo e title para título da imagem.

  5. Formatação de Tabelas

    6. HTML permite criar tabelas usando os elementos table, tr para linhas, td para células de dados e th para cabeçalhos de tabela. Você pode usar atributos como colspan e rowspan para mesclar células.

  6. Formulários

    7. HTML fornece elementos para criar formulários, como form, input, select, textarea, button, entre outros. Estes elementos permitem a coleta de informações do usuário.

  7. Formatação de Layout

    8. Embora a formatação de layout seja geralmente realizada com CSS, HTML também possui elementos para ajudar na estruturação do layout, como div e span, que são usados para criar contêineres e agrupar elementos para estilização e posicionamento.

Quotations

Em HTML, as citações (quotations) são usadas para incluir texto citado dentro de um documento HTML. Existem dois elementos principais que são usados para representar citações em HTML: blockquote e q.

  1. Blockquote

    2. Este elemento é usado para representar um bloco de texto citado de outra fonte. Ele geralmente é exibido com uma margem à esquerda e/ou à direita para destacá-lo do restante do conteúdo. Exemplo:

                               <p>Texto introdutório.</p>
                           <blockquote>
                                <p>Este é um bloco de texto citado de outra fonte.</p>
                           </blockquote>
                           <p>Texto após a citação.</p>

    Texto introdutório.

    Este é um bloco de texto citado de outra fonte.

    Texto após a citação.

  2. Tag q

    3. Este elemento é usado para representar uma citação curta dentro do texto. Por padrão, navegadores costumam envolver a citação com aspas.Exemplo:

                                <p>Alguns dizem que <q>o silêncio é de ouro</q>.</p>

    Alguns dizem que o silêncio é de ouro.

Além desses elementos, você pode usar atributos adicionais para fornecer informações contextuais sobre as citações, como cite, que especifica a URL da fonte da citação.

                    <blockquote cite="https://www.example.com/citacao">
                        <p>Este é um bloco de texto citado de outra fonte.</p>
                    </blockquote>

Este é um bloco de texto citado de outra fonte.

Em resumo, em HTML, as citações são representadas usando os elementos blockquote e q, permitindo que os desenvolvedores destaquem e identifiquem o texto citado dentro de um documento HTML.

Block & Inline

Em HTML, elementos são classificados como "bloco" (block) ou "em linha" (inline), dependendo de como eles são exibidos na página da web e de como eles interagem com outros elementos. Essa classificação afeta o layout e o comportamento dos elementos na página.

  1. Elementos de Bloco (Block Elements):

    2. Os elementos de bloco ocupam todo o espaço horizontal disponível em seu contêiner pai e começam em uma nova linha. Eles geralmente são usados para estruturar o layout da página e incluem elementos como div, p, h1 a h6, ul, ol, li, table, form, entre outros.

  2. Elementos em Linha (Inline Elements):

    3. Os elementos em linha são exibidos na mesma linha que o conteúdo que os rodeia e ocupam apenas o espaço necessário para o conteúdo. Eles são frequentemente usados para marcar partes do texto e incluem elementos como span, a, strong, em, img, input, button, entre outros.

Aqui está um exemplo simples que ilustra a diferença entre elementos de bloco e em linha:

            <h1>Elemento de Bloco</h1>
                <p>Este é um parágrafo de texto dentro de um elemento de bloco.</p>
            </div>

            <style>
            .block {
                background-color: lightblue;
                border: 1px solid blue;
                margin-bottom: 10px;
            }

Iframes

Um iframe é um elemento HTML que permite incorporar outro documento HTML dentro de uma página web. A palavra "iframe" significa "inline frame" (quadro em linha), e esse elemento cria uma área retangular na página onde outro documento HTML pode ser exibido.

Os iframes são frequentemente usados para incorporar conteúdo de outros sites em uma página da web. Por exemplo, um vídeo do YouTube, um mapa do Google Maps ou um widget de mídia social podem ser incorporados em uma página usando um iframe.

Aqui está um exemplo simples de uso de um iframe para incorporar uma página da web:

            <h3>Conteúdo da Página Principal</h3>
            <p>Aqui está o conteúdo da página principal.</p>
        
            <h3>Conteúdo Incorporado</h3>
            <iframe src="https://www.disney.com" width="600" height="400" title="Exemplo"></iframe>

Semantics

A semântica em HTML diz respeito à correta utilização dos elementos HTML para comunicar sentido e organização ao conteúdo da página web. Isso implica na seleção dos elementos HTML mais apropriados para descrever o tipo de conteúdo que estão sendo marcados, tornando o código HTML mais claro e fácil de entender para os desenvolvedores, além de mais acessível tanto para os usuários quanto para os mecanismos de busca.

Alguns exemplos são: header, main, section, nav, aside, footer e etc

Ao utilizar a semântica correta em HTML, você torna seu código mais claro e legível, melhora a acessibilidade para todos os usuários e ajuda os mecanismos de busca a entender e indexar melhor o conteúdo da sua página. Isso pode levar a uma melhor classificação nos resultados de pesquisa e uma experiência geral mais agradável para os usuários.

Entities

Entidades HTML são códigos especiais utilizados para representar caracteres especiais, como símbolos, acentos e caracteres que têm significados especiais em HTML. Esses códigos são usados porque alguns caracteres têm significados especiais em HTML e podem ser interpretados de forma incorreta pelo navegador se forem usados diretamente no código HTML.

Por exemplo, o sinal de maior que (>) é interpretado pelo navegador como o fechamento de uma tag HTML. Se você quiser exibir o sinal de maior que em seu documento HTML sem que ele seja interpretado como uma tag, você pode usar a entidade HTML correspondente, que é &lt;.

Aqui estão alguns exemplos de entidades HTML comuns:

  1. &lt;: Menor que (<)
  2. &gt;: Maior que (>)
  3. &amp;: E comercial (&)
  4. &quot;: Aspas duplas (")
  5. &apos;: Aspas simples (')

Charsets

Charsets, ou conjuntos de caracteres, são conjuntos de códigos numéricos que representam caracteres em uma linguagem específica. Em HTML, o charset (conjunto de caracteres) é uma meta informação que especifica qual codificação de caracteres deve ser usada para interpretar o conteúdo do documento HTML.

A meta tag meta charset="charset-name" é usada no cabeçalho (header) do documento HTML para definir o charset. Por exemplo, meta charset="UTF-8" especifica que o documento HTML está codificado usando UTF-8, que é uma codificação de caracteres que abrange a maioria dos caracteres usados em praticamente todos os idiomas escritos.

Alguns dos conjuntos de caracteres mais comuns usados em HTML incluem:

  1. UTF-8: É o padrão recomendado para páginas da web modernas. Ele suporta uma vasta gama de caracteres e é amplamente compatível com diferentes idiomas e scripts.
  2. ISO-8859-1: Também conhecido como Latin-1, é um conjunto de caracteres comum que suporta a maioria dos idiomas europeus.
  3. Windows-1252: É uma extensão do ISO-8859-1 que inclui caracteres adicionais para idiomas ocidentais.

Ao definir o charset corretamente em um documento HTML, você garante que os navegadores interpretem o texto corretamente e exibam os caracteres conforme esperado. Se o charset não estiver definido corretamente, os navegadores podem interpretar o texto de maneira incorreta, resultando em caracteres estranhos ou ilegíveis na página.

            <meta charset="UTF-8">

HTML vs XHTML

HTML (HyperText Markup Language) e XHTML (eXtensible HyperText Markup Language) são duas versões da linguagem de marcação usada para criar páginas da web. Embora tenham uma semelhança superficial e compartilhem muitas características comuns, existem algumas diferenças importantes entre HTML e XHTML:

Sintaxe

  • HTML: Em HTML, a sintaxe é mais flexível. Os elementos podem ser escritos em maiúsculas ou minúsculas, as tags não precisam ser fechadas e os atributos não precisam estar entre aspas.
  • XHTML: Em XHTML, a sintaxe é mais rigorosa e é uma versão mais restrita do XML. As tags devem ser todas em minúsculas, devem ser fechadas corretamente e os atributos devem estar entre aspas.

Documentação

  • HTML: O HTML tem uma documentação mais ampla e estabelecida, com muitos recursos e tutoriais disponíveis online.
  • XHTML: O XHTML é baseado em XML e segue regras mais estritas de formatação e validação. Ele é considerado mais limpo e bem-estruturado, mas pode ser mais difícil de aprender para iniciantes devido à sua sintaxe mais rigorosa.

Compatibilidade

  • HTML: O HTML é mais amplamente suportado pelos navegadores e é usado em uma variedade de contextos de desenvolvimento web.
  • XHTML: Embora o XHTML tenha sido proposto como a evolução natural do HTML, sua adoção não foi tão generalizada quanto o HTML. Além disso, algumas versões de XHTML foram descontinuadas ou não receberam mais suporte.

Aplicações

  • HTML: O HTML é mais comumente usado para páginas da web tradicionais, especialmente em contextos onde a compatibilidade com navegadores mais antigos é uma preocupação.
  • XHTML: O XHTML é frequentemente usado em ambientes onde a validação estrita e a compatibilidade com outras tecnologias XML são importantes, como em aplicativos baseados em XML ou em ambientes de desenvolvimento móvel.

Em resumo, HTML e XHTML são duas versões da mesma linguagem de marcação usada para criar páginas da web, cada uma com suas próprias características e usos distintos. A escolha entre HTML e XHTML dependerá das necessidades específicas do projeto e das preferências do desenvolvedor.

Forms

Formulários HTML (HTML forms) são elementos que permitem aos usuários interagir com uma página da web, enviando dados para o servidor web. Eles são usados para coletar informações do usuário, como texto, opções selecionadas, uploads de arquivos e outros tipos de dados.

Um formulário HTML é definido usando a tag form, e pode conter vários elementos de entrada, como campos de texto, caixas de seleção, botões de rádio, botões de opção, botões de envio e muito mais. Quando um usuário envia um formulário, os dados são enviados para o servidor web especificado na ação do formulário.

Exemplo:

            <form action="/processar-formulario" method="post">
                <label for="nome">Nome:</label>
                <input type="text" id="nome" name="nome" required>
                
                <label for="email">E-mail:</label>
                <input type="email" id="email" name="email" required>
                
                <label for="mensagem">Texto:</label>
                <textarea id="mensagem" name="mensagem" rows="4" required></textarea>
                
                <button type="submit">Enviar</button>
            </form>

Form Attributes

Atributos de formulário HTML são atributos que podem ser usados em elementos form e elementos de entrada (input, textarea, select, etc.) para controlar o comportamento e a funcionalidade dos formulários HTML. Esses atributos ajudam a definir como os dados do formulário são coletados, processados e enviados.

Aqui estão alguns exemplos:

  • action
  • method
  • disabled
  • id
  • value
  • name
  • placeholder

Exemplo de um formulário HTML simples com alguns atributos comuns:

            <form action="/enviar-dados" method="post">
                <label for="nome">Nome:</label>
                <input type="text" id="nome" name="nome" required><br>
                
                <label for="email">E-mail:</label>
                <input type="email" id="email" name="email" required><br>
                
                <label for="mensagem">Mensagem:</label>
                <textarea id="mensagem" name="mensagem" rows="4" required></textarea><br>
                
                <button type="submit">Enviar Mensagem</button>
            </form>



O formulário é enviado para o URL "/enviar-dados" usando o método POST form action="/enviar-dados" method="post". Existem três campos no formulário: Nome, E-mail e Mensagem, cada um com um label correspondente e um campo de entrada. O campo "Nome" é obrigatório (required) e é definido como input type="text" id="nome" name="nome" required. O campo "E-mail" também é obrigatório e deve ser um e-mail válido (type="email"). O campo "Mensagem" é uma área de texto textarea que permite ao usuário inserir uma mensagem mais longa.

Javascript

JavaScript é uma linguagem de programação de alto nível, interpretada, orientada a objetos e multi-paradigma. Foi inicialmente desenvolvida para ser executada em navegadores web para interagir dinamicamente com o conteúdo de páginas HTML, mas desde então se expandiu para várias outras áreas, incluindo desenvolvimento web no lado do servidor (Node.js), desenvolvimento de aplicativos de desktop e móveis, e até mesmo desenvolvimento de jogos.

Aqui estão alguns pontos-chave sobre JavaScript:

  • Interatividade em Páginas Web: JavaScript é amplamente usado para criar interatividade em páginas web. Ele pode ser usado para manipular elementos HTML, responder a eventos do usuário (como cliques de mouse e pressionamentos de tecla), validar formulários, fazer requisições para servidores e atualizar o conteúdo de uma página sem recarregá-la completamente (AJAX).
  • Linguagem de Script do Lado do Cliente: JavaScript é uma linguagem de script do lado do cliente, o que significa que o código é executado no navegador do usuário, não no servidor. Isso permite uma experiência mais rápida e dinâmica para os usuários, pois não há necessidade de fazer solicitações adicionais ao servidor para atualizar o conteúdo da página.
  • Sintaxe Simples e Flexível: JavaScript possui uma sintaxe simples e flexível, que é fácil de aprender para iniciantes e poderosa o suficiente para lidar com projetos complexos. Sua sintaxe é semelhante à de outras linguagens de programação, como Java e C, o que facilita a transição para desenvolvedores que já têm experiência em outras linguagens.
  • Ecossistema Rico: JavaScript tem um ecossistema muito rico, com uma grande variedade de bibliotecas e frameworks disponíveis para facilitar o desenvolvimento web. Alguns dos mais populares incluem jQuery, React.js, Angular, Vue.js e Node.js.
  • Orientada a Eventos: Uma das características mais importantes do JavaScript é sua natureza orientada a eventos. Os desenvolvedores podem escrever código para responder a uma variedade de eventos, como cliques de mouse, pressionamentos de tecla, submissões de formulários e carregamento de página.

JavaScript é uma parte essencial do desenvolvimento web moderno e é amplamente usado em conjunto com HTML e CSS para criar experiências interativas e dinâmicas para os usuários da web.
